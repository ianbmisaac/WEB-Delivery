# rq-10: Funcionalidad avanzada del dominio — Pedidos delivery

## Actor

Usuario autenticado

## Precondiciones

Login operativo (GEN-05) y modelos del dominio definidos.

## Postcondiciones

El dominio **Pedidos delivery** soporta: asignar repartidor y ETA simulado. Evidencia en UI y API.

## Descripción detallada

Asignar repartidor y eta simulado.

El alumno define nombres de tablas, campos y rutas REST. La necesidad de negocio debe quedar cubierta sin copiar esquemas de otras evaluaciones del curso.

## Requiere página web integrada

Parcial — pantalla de reporte, integración externa o cierre según el caso

## Criterios de aceptación (checklist)

- [ ] Implementado en API según necesidad del dominio
- [ ] Pantalla web asociada operativa contra API desplegada o local
- [ ] Documentado en README o matriz de avance

## Libertad de diseño (qué decide el alumno)

- Nombre de modelos Sequelize y migraciones
- Campos opcionales y relaciones (1:N, N:M)
- Diseño de pantallas y navegación
- Textos, colores y layout del front
